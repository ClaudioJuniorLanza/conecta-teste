Login with Imaginacom

Um plugin simples e rápido para se logar utilizando as redes sociais.


Manual:
Google: https://developers.google.com/identity/sign-in/web/reference
Facebook: https://developers.facebook.com/docs/facebook-login/manually-build-a-login-flow

Modo de uso

Inclua a API Javascript do loginWithImaginacom, e utilize o método .create() para renderizar o iframe.

<script type='text/javascript' src='https://ws.imaginacom.com/login/api.js'></script>
<script type='text/javascript'>
  var lwi = loginWithImaginacom.create(false, function(provider, token){ /* Login bem sucedido */ });
</script>
-- ou --
<div id='myDiv'></div>
<script type='text/javascript'>
loginWithImaginacom.create('myDiv', {
	token:       none,              // --> Opcional. Token da IMAGINACOM para customizar o AppId e AuthCode. A customização é feita no painel Gestão da IMAGINACOM.
	display:     none,              // --> Opcional, mantém apenas a API disponível;
	providers:   'facebook,google', // --> Opcional. Se não tiver uma lista, ativa tudo;
	checked:     'facebook,google', // --> Muda o ícone para 'checked' nos provedores informados;
	autoCheck:   true,              // --> Muda ícone para "checked" quando houver um login bem sucedido;
	autoLoading: true,              // --> Alterna para "loading" enquanto a popup estiver aberta;
	onLogin:     function(provider, token, rawData){}
	onLoginFail: function(provider, rawData){} // Opcional
	onUncheck:   function(provider){}          // Opcional
});
</script>
Copy & Paste: 
<script type='text/javascript' src='https://ws.imaginacom.com/login/api.js'></script>
<script type='text/javascript'>
loginWithImaginacom.create(false, {
	checked: '',
	onLogin: function(provider, token){
		// Faça a validação server-side do token.
		return true; // Retorne "true" para interromper a animação. Ou utilize this.setChecked(true|false) depois.
	},
	onUncheck: function(provider){
		// Implemente este método para processar "uncheck".
		// Deixe em branco para utilizar o padrão de onLogin() ou onLoginFail()
		return true; // Retorne "true" para interromper a animação. Ou utilize this.setChecked(true|false) depois.
	},
});
</script>

Métodos da API (loginWithImaginacom)

.create(divId, options) ou .create(divId, onLogin)

divId:
ID do Div que receberá o rame, ou NULL para criar no local atual.

options:
token:
Opcional.		 (null) | (string)
Padrão: null 		Token da IMAGINACOM para customizar o AppId e AuthCode.
A customização é feita no painel Gestão da IMAGINACOM.
display:
Opcional.		 (null) | 'none'
Padrão: null		 Não exibe o iframe, só disponibiliza os métodos
providers:
Opcional.		 facebook,google
Padrão: null		 Restringir aos provedores informados.
Se não informado, exibirá tudo que tiver disponível.
checked:
Opcional.		 facebook,google
Padrão: null		 Altera o estado inicial dos provedores informados para "checked"
autoCheck
Opcional.		 true | false
Padrão: true		 Após o login bem sucedido, muda o ícone para "checked"
autoLoading
Opcional.		 true | false
Padrão: true		 Após abrir a popup, muda para "loading" até a popup ser fechada.
onLogin: callback(provider, token, rawData)
    // Provider pode ser 'google' ou 'facebook'
    // Envie o token server-side para https://ws.imaginacom.com/login/checkAccessToken.php?provider=xxx&token=yyy
    // Retorne TRUE para interromper o Loading e mudar para "checked" (se autoCheck=true)
onLoginFail:
Opcional.		 callback(provider, rawData)
    // Provider pode ser 'google' ou 'facebook'
    // Exemplos de rawData constam no final deste documento.
    // Opcional: Se não informado, mostrará alert() padrão.
    // Retorne TRUE para interromper o Loading.

Facebook: { authResponse: undefined, status: undefined }
Google:   { error: "popup_closed_by_user" | "access_denied"  | "immediate_failed" }
onUncheck:
Opcional.		 callback(provider)
    // Provider pode ser 'google' ou 'facebook'
    // Opcional: Se não informado, "onLogin" ou "onLoginFail" serão chamados.
    // Retorne RUE para interromper o Loading e mudar para "checked"

.setChecked(provider, yesno)

Muda o ícone do provider em questão, para true=checked ou false=normal.
Envie 'loading' para animação de "carregando".

.request(provider, method, params, callback)

Chama os métodos disponíveis no Javascript Framework do provedor.
var instance = loginWithImaginacom.create(...);
instance.request('facebook', "FB.getLoginStatus", function(authData){ console.log("Resultado de authData", authData) });
instance.request('facebook', "FB.api", "/me?fields=id,name,email" function(response){ ... });

Validação server-side:

Validação rápida

O método "onLogin" vai te retornar o TOKEN, a ser utilizado na validação server-side.
$checkUrl = "https://ws.imaginacom.com/login/checkAccessToken.php?provider={$provider}&access_token={$token}";
if($TOKEN_IMAGINACOM){
  // Se você utilizou um token da imaginacom em .create, para customizar a tela de Login, utilize-o aqui também.
  $checkUrl .= "&token={$TOKEN_IMAGINACOM}";
}
$userData = json_decode(file_get_contents($checkUrl));

Validação avançada/manual do acebook:

Antes de utilizar um token, é importante validar se ele pertence ao nosso aplicativo.
Para isso é importante o chamado a /checkAccessToken.php, que vai retornar erro caso não seja.
$fb = dFacebook::start('iPainel');

// Se você não quiser verificar o /checkAccessToken embutido, você precisará de um app_id e um app_secret:
if($useLoginApi){
	$userData = json_decode(file_get_contents("https://ws.imaginacom.com/login/checkAccessToken.php?provider=facebook&access_token=$token"), true);
}
else{
	$fb->setConfig($APP_ID, $APP_SECRET);
	$userData = $fb->debugToken($token);
	// O retorno será idêntico ao descrito em "Validação Rápida > Facebook On Success"
	// Verifique se $userData[app_id] é válido.
}

// Você também já tem $userData[user_id] disponível.
// Vamos começar a trabalhar com o token:
$fb->setAccessToken($token);
$fb->sessionUpdate()  (Para retornar ->isLogged() true no futuro)
